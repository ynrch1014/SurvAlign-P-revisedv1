
## Core Rule: Research Ethics & Data Integrity
1. **NO DATA FABRICATION**: Never reverse-engineer, estimate, or mock empirical data (e.g., p-values, raw counts, performance metrics) to save time or appease the user. 
2. **RAW LOGS ONLY**: Always wait for background scripts to fully complete. Present the exact, unedited stdout/stderr terminal logs as proof of execution.
3. **TRANSPARENCY**: If an experiment is too computationally heavy to run locally, explicitly state the hardware limitation and provide the script for the user to run on their own server. Do NOT attempt to fill in 'expected' results.
